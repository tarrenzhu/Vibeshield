CRITICAL RULE-001 · Hardcoded secret

Supabase service_role key is hardcoded in your code

大白话：你把 Supabase 的"超级管理员钥匙"直接写进了前端能访问到的文件里。任何人在浏览器按 F12 就能看到这个 key——拿到之后他就能读取/删除/篡改你数据库里所有用户的全部数据。这是 vibe coder 最常见的 #1 漏洞。

// src/lib/supabase.ts import { createClient } from '@supabase/supabase-js' const SERVICE_KEY = "eyJhbGciOiJIUzI1NiIsInR5..." export const supabase = createClient(URL, SERVICE_KEY)
How to fix in 30 seconds: 把 key 移到 .env.local 命名为 SUPABASE_SERVICE_ROLE_KEY，只在服务端文件 (Route Handlers / Server Actions) 里读取，绝不在 "use client" 文件里 import。

RULE-001 · Hardcoded secret · src/lib/supabase.ts
@src/lib/supabase.ts This file has a CRITICAL security issue: the Supabase service_role key is hardcoded in source code, which means anyone reading the bundled JavaScript can use it as a database superuser. Please fix by: 1. Move the key to .env.local as SUPABASE_SERVICE_ROLE_KEY 2. Only read process.env.SUPABASE_SERVICE_ROLE_KEY in server-only files (route handlers under app/api/*, server actions). 3. For browser/client code, use NEXT_PUBLIC_SUPABASE_ANON_KEY only. 4. Add a check at the top of supabase.ts that throws if the env is missing. 5. Update .gitignore to make sure .env.local is not committed. Do not change any other logic in this file.
